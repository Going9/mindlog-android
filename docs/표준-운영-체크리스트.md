# Mindlog Android 표준 운영 체크리스트 (Hotwire Native Android)

## 1. 목적
- 브랜치: `release/android-starter-v1`
- 목표: Hotwire Native 표준 구성을 유지하고, 네이티브 커스텀을 최소화한다.
- 범위: `로그인 복귀 + 홈 + 일기 CRUD 네비게이션`.

## 1-1. 브랜치 운영 규칙
- 구현은 작업 브랜치에서만 수행한다. 브랜치 prefix 표준: `feat/*`, `fix/*`, `refactor/*`, `docs/*`, `test/*`, `chore/*`, `perf/*`, `build/*`, `ci/*`, `hotfix/*`.
- 작업 브랜치는 `release/android-starter-v1`로 먼저 머지한다.
- 릴리즈 검증 후 `release/android-starter-v1`를 `main`으로 머지한다.
- `release/android-starter-v1`, `main`에서 직접 기능 수정 커밋을 금지한다.
- 커밋 메시지는 Conventional Commits prefix(`feat:`, `fix:`, `refactor:`, `docs:`, `test:`, `chore:`, `perf:`, `build:`, `ci:`, `revert:`) + 한국어 제목을 사용한다.

## 2. 표준 기준(잠금값)
- 라이브러리: `dev.hotwire:core`, `dev.hotwire:navigation-fragments`.
- 앱 초기화: `Application`에서 1회 수행.
- 네비게이션: `HotwireActivity` + 기본 Navigator 중심.
- 경로 규칙: path configuration 중심으로 처리.
- 외부 로그인: `/auth/login.*`만 외부 브라우저 오픈.

## 3. 공식 레퍼런스

- Android Configuration: https://native.hotwired.dev/android/configuration
- Android Path Configuration: https://native.hotwired.dev/android/path-configuration
- Basic Navigation: https://native.hotwired.dev/overview/basic-navigation

## 5. 금지 규칙(재발 방지)
- 화면 깜빡임 대응을 위해 커스텀 Fragment/전환 애니메이션 파일을 계속 추가하는 방식 금지.
- Activity에 초기화/인증/네비게이션/상태관리 책임을 과도하게 누적하는 방식 금지.
- path configuration 바깥에서 URL 예외처리를 임의 확장하는 방식 금지.
- `BASE_URL` 하드코딩 금지.

## 6. 권장 규칙(표준 근접)
- 시작 URL은 `/?source=app`으로 유지해 서버 세션 컨텍스트를 일관되게 만든다.
- 딥링크 토큰 교환 URL은 인코딩 후 1회 처리한다.
- 중복 딥링크 이벤트는 dedup 로직으로 차단한다.
- 진행 표시 UI는 Hotwire 기본 레이아웃과 큰 차이 없이 유지한다.

## 7. 환경 규칙(dev/prod)
- dev URL은 `MINDLOG_DEV_BASE_URL` Gradle property로 주입한다.
- 에뮬레이터 기본값 권장: `https://10.0.2.2:8443`
- 실기기 adb reverse 사용 시: `https://localhost:8443`
- prod URL: `https://www.mindlog.blog`

- [ ] 로그인 직후 홈에서 인증 UI가 즉시 반영되는가?
- [ ] 리스트 -> 상세 이동 시 슬라이드 잔상/흰 화면 플래시가 재현되지 않는가?
- [ ] 로그아웃/세션 만료 후 재로그인 플로우가 깨지지 않는가?
- [ ] `./gradlew test` 통과 + 핵심 수동 E2E(로그인/CRUD) 통과인가?

## 9. 운영 메모(이번 이슈 기준)
- 간헐적 흰 화면/스피너는 서버 비즈니스 오류보다 WebView 전환 타이밍 영향이 큰 증상이다.
- 해결 순서는 `표준 네비게이션 고정 -> 중복 방문 제거 -> 최소 커스텀 유지`로 진행한다.
